基础知识

在复现题目之前，我们可以了解一下断点的类型以及不同的用处

1.软件断点

原理：调试器把目标指令换成int3/0xCC,程序执行到这里就中断

用途：

场景 用法
看某个函数是否被调用 下在函数入口
分析关键判断 下在 cmp / test / jne / je 附近
patch 前确认逻辑 断在分支前看寄存器
找密码校验 断在 strcmp / memcmp / strncmp

优点：简单、数量基本不限
缺点：会修改代码段，容易被反调试检测，比如检查 0xCC 或代码校验和

2.硬件断点

利用CPU调试寄存器，比如x86/x64的DR0-DR3,不修改目标代码

常见类型：

类型 触发条件 用途
执行断点 执行到某地址 替代软件断点，绕过代码校验
写入断点 某地址被写 找谁修改了变量
读取断点 某地址被读 找谁使用了关键数据
访问断点 读或写某地址 追踪敏感内存

当程序调试时存在CRC32冗余效验时，我们下软件断点会调试不了，这时我们可以尝试使用硬件断点

优点：不改代码，适合反调试场景
缺点：数量少，通常只有 4 个；也可能被检查调试寄存器发现

3.内存断点

原理：对一整页内存设置访问保护，程序访问该页时触发异常

用途：

场景 用法
不知道具体变量地址 对一大片内存下访问断点
追踪解密后的代码执行 对新分配内存下执行断点
壳/自解密程序 断在 unpack 后的代码页
找字符串什么时候被读 对字符串所在页下读断点

缺点：粒度是页，容易频繁触发，噪声比较大

4.条件断点

断点触发时先判断条件，条件满足才真正停下。

例子：

用途：

场景 用法
循环次数太多 只在某次循环停
函数被大量调用 只在参数符合时停
过滤无关输入 只在 buffer 指向用户输入时停
爆破/校验逻辑 只在比较失败/成功时停

比如 memcmp 被调用很多次，可以只在参数指向你输入的字符串时停

5.异常断点
程序发生异常时停下，比如：

  • 访问违规
  • 除零
  • 非法指令
  • int3
  • 单步异常
  • C++ 异常
  • SEH 异常

用途：

场景 用法
反调试利用异常流 看异常处理逻辑
VM/壳用异常跳转 追踪真实控制流
程序故意触发崩溃 判断是不是保护逻辑
int3 反调试 分析异常是否被程序自己处理

有些题会用 SEH/VEH 控制流程，正常静态看很乱，异常断点能直接抓到跳转点

6.API 断点
断在系统 API 或库函数上。

CTF 逆向里特别实用：

API / 函数 用途
strcmp, memcmp, strncmp 找密码比较
scanf, gets, fgets, read 找输入位置
printf, puts 找输出位置
malloc, VirtualAlloc 找动态内存
VirtualProtect, mprotect 找壳、自修改代码
LoadLibrary, GetProcAddress 找动态导入
CreateFile, ReadFile, fopen 找文件校验
IsDebuggerPresent, CheckRemoteDebuggerPresent, ptrace 反调试
recv, send 网络题流量逻辑

经典做法：不知道主逻辑在哪，就先断输入、比较、输出、内存分配、权限修改这些关键 API

  1. 日志断点 / Trace 断点
    触发时不暂停，只打印寄存器、参数、内存值，然后继续运行。

用途：

  • 记录函数参数
  • 追踪加密轮函数状态
  • 看循环中 key/state 怎么变化
  • 不想频繁手动继续运行

适合分析算法题，比如 TEA、RC4、AES 变种、VM opcode 分发

复现题目

依旧典型C/S的re题目

先查壳工具分析题目，都没有加壳

打开分析主函数发现有控制流混淆，想用D810插件去除，没成功

我们直接分析string，看看有没有有用的API，发现了Bcrypt***函数，这是windows自带的加解密函数库

下上断点开始调试这几个api函数，然后我们可以在寄存器窗口或者右下角stack窗口，可以看到几个标志

AES,MD5,RC4这三个算法

很明显最开始让我们输入username然后对我们的用户名先进行rc4加密，再进行md5

想找到比对的地方，对那个0x012345789ABCDEF字符串下断点，这是md5的常量特征

然后不断F8，很快就可以看到我们的输入的md5值，会进行比较，那我们直接下一个硬件断点进行调试，最终找到正确的md5值，然后在线网站直接解决

拿到我们的正确用户名：ctfer

拿到用户名之后，我们可以多调试几遍去了解程序流程

serial 先经过 RC4 传输
server 还原出 serial 明文
server 用 AES-CBC 相关逻辑校验 serial
如果匹配，就生成 “OK: welcome ctfer”
最后把这条响应再用 AES-CBC 加密发回 client

在我们的输入上下硬件断点然后单步进行调试可以找到和用户名那有点像的对比逻辑
